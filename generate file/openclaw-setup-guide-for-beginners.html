一、安装 Node.js（必须先装这个）

OpenClaw 是基于 Node.js 运行的工具，就像汽车需要发动机一样，必须先安装 Node.js 才能安装 OpenClaw。这一步不能跳过。

第一步：下载 Node.js

  1. 打开浏览器，访问：https://nodejs.org/
  2. 点击页面中央的绿色大按钮 "LTS"（长期支持版，最稳定）
  3. 等待下载完成，下载的文件是 .pkg 格式（约 30MB）
如果你看到两个按钮，选左边那个"LTS"版本，不要选右边带版本号的测试版。

第二步：安装 Node.js

  1. 双击下载的 .pkg 文件
  2. 点"继续"→"同意"→"安装"
  3. 输入你的 Mac 登录密码
  4. 等待进度条完成，点"安装完成"

第三步：验证安装成功

  1. 按 Command + 空格键，搜索"终端"，回车打开
  2. 在终端里分别输入以下两行命令，每行输完后按回车：
node -v
npm -v
✅ 成功的标志
如果看到类似 v20.x.x 或 v18.x.x 的版本号（两行都要有），说明安装成功！
---

二、安装 OpenClaw

第一步：打开终端

按 Command + 空格键，搜索"终端"，回车

第二步：运行安装命令

在终端里粘贴（Command + V）这行命令，然后回车： 
npm install -g openclaw
安装过程中如果提示输入密码，输入你的 Mac 登录密码（输入时屏幕不显示任何字符，正常现象），然后回车。

第三步：等待安装完成

- 终端会开始下载，需要等待 1-3 分钟 - 看到类似 added xxx packages 的字样就是装好了 - 出现红色 ERROR 字母才是出错，只显示进度条不代表出错

第四步：验证安装成功

openclaw --version
看到版本号（如 🦞 OpenClaw 2026.4.15）就说明装好了 ✅ ---

三、启动 OpenClaw 并注册账号

第一步：启动网关

在终端运行： 
openclaw gateway run
这一步不能关闭窗口！关闭窗口 Gateway 就停止了，就像关掉电视一样。

第二步：看到类似这样的输出

🦞 OpenClaw 2026.4.15 (041266a)
Gateway: running, port 18789
Dashboard: http://localhost:18789/

第三步：打开浏览器访问控制台

  1. 打开 Chrome 或 Safari
  2. 地址栏输入：http://localhost:18789
  3. 第一次打开会让你注册账号、设置密码
  4. 按提示完成注册
注册完成后，建议把这个地址保存到浏览器书签，方便下次快速访问。
---

四、配置 QQBot（接收 QQ 消息）

⚠️ 前提条件
这一步需要先有 QQ 开放平台的开发者账号。如果没有，建议先跳过，用网页版和控制台先玩起来，避免卡在这里。

QQ 开放平台申请：https://q.qq.com（需要审核，有一定难度）

如果你已经有了 QQ 开放平台的 AppID 和 Token，按以下步骤：

第一步：安装 QQBot 插件

openclaw plugins install qqbot

第二步：配置 AppID

openclaw config set plugins.entries.qqbot.config.appId 你的AppID

第三步：配置 Token

openclaw config set plugins.entries.qqbot.config.token 你的Token

第四步：重启网关使配置生效

openclaw gateway restart

第五步：在 QQ 开放平台配置回调地址

在 QQ 开放平台的机器人配置页面，填入你的回调地址（需要先有公网域名，新手较难实现）

五、配置 Web 控制台访问（bind 设置）

bind 就像一扇门，决定谁能进入 OpenClaw 控制台。这一步配置好之后，同一局域网内的其他设备（比如手机、平板、其他电脑）都能访问你的控制台。

bind 的三种模式

模式谁能访问推荐场景
local只有安装 OpenClaw 的这台电脑默认，不推荐
lan同一局域网的所有设备 ✅最常用，推荐！
tailnet通过 Tailscale VPN 访问需要额外配置 VPN

第一步：查看当前 bind 模式

openclaw config get gateway.bind

如果显示 local，需要改成 lan

第二步：修改 bind 为 lan

openclaw config set gateway.bind lan

第三步：重启网关

openclaw gateway restart

第四步：查看你的局域网 IP 地址

openclaw gateway status
在输出里找类似这样的地址：
Dashboard: http://192.168.1.112:18789/
（`192.168.1.112` 是示例，你的可能不一样）

第五步：从其他设备访问

  1. 确保设备（手机/平板/其他电脑）和 Mac 在同一个 Wi-Fi 网络
  2. 打开浏览器
  3. 地址栏输入：http://192.168.1.112:18789（把 IP 换成你查到的）
把查到的 IP 地址发送给同事，他们就能在自己的电脑上访问你的 OpenClaw 控制台了！

六、SSH 隧道方法（解决浏览器安全限制）

🚨 什么情况下需要这个？
从局域网另一台电脑访问时，浏览器报错：
control ui requires device identity (use HTTPS or localhost secure context)

这是浏览器的安全策略，局域网 IP 被认为是"不安全"的，需要用 SSH 隧道绕过。

SSH 隧道原理

SSH 就像一条秘密隧道，把远程电脑的端口"传送"到你本地电脑上。这样访问远程服务时，浏览器以为是访问本地，就不会有安全限制了。

操作步骤（从 192.168.1.109 访问 192.168.1.112 上的 OpenClaw）

第一步：在 192.168.1.109 这台电脑上打开终端

第二步：运行 SSH 隧道命令

ssh -L 18789:localhost:18789 omi@192.168.1.112
把 omi 换成你的 Mac 用户名（打开 Mac 系统偏好设置 → 用户与群组可以看到）
把 192.168.1.112 换成你的 Mac 的局域网 IP 地址

第三步：输入密码

终端会提示输入密码。输入你的 Mac 登录密码后回车（输入时屏幕不显示字符，正常）。

第四步：浏览器访问

登录成功后，在同一终端窗口保持运行，然后新开一个浏览器标签页，地址栏输入：

http://localhost:18789
✅ 成功！
现在可以在浏览器正常使用 OpenClaw 控制台了。
终端窗口不能关闭！关闭的话隧道就断了。

七、常见问题

问题原因解决方法
终端提示 Permission denied 权限不够 在命令前加 sudo，如：sudo npm install -g openclaw
gateway run 报错 port already in use 18789 端口被占用 换端口：openclaw config set gateway.port 19898 然后重启
重启后 bind 又变回 local 配置被重置 重新运行：openclaw config set gateway.bind lan
QQBot 收不到消息 插件/配置问题 确认 gateway 在运行 + AppID 和 Token 正确 + 重启 gateway
openclaw: command not found 安装没成功 重新运行：npm install -g openclaw
浏览器空白或报错 Gateway 没启动 确认终端里 gateway 在运行，没有报错信息
