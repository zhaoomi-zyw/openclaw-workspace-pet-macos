二、Agent 基础配置

2.1 安装 OpenClaw

# macOS / Linux 全局安装
npm install -g openclaw

# 启动网关
openclaw gateway run

# 检查状态
openclaw gateway status

2.2 创建 Agent

# 创建 agent（pet-macos 为示例名称）
openclaw agents create pet-macos

# 切换 agent
openclaw agents use pet-macos

2.3 连接 QQBot 消息通道

# 安装 QQBot 插件
openclaw plugins install qqbot

# 配置（需到 QQ 开放平台申请 AppID + Token）
openclaw config set plugins.entries.qqbot.config.appId YOUR_APP_ID
openclaw config set plugins.entries.qqbot.config.token YOUR_TOKEN

3.3 Tavily API Key 配置

# 1. 到 https://tavily.com 注册（开发版免费）
# 2. 获取 API Key
# 3. 写入 ~/.openclaw/.env
echo 'TAVILY_API_KEY=tvly-your-key-here' >> ~/.openclaw/.env

# 4. 验证
source ~/.openclaw/.env
curl -X POST "https://api.tavily.com/search" \
  -H "Authorization: Bearer $TAVILY_API_KEY" \
  -d '{"query":"test","max_results":1}'
# 正常返回 JSON 结果即为成功

OpenClaw 每个会话都是"全新醒来"的，AI 没有持续的自我记忆。所有积累必须写进文件，文件 = AI 的大脑。

五、网络搜索规范（必读）

🔴 信息来源优先级
Tavily → CDP → 放弃/替代（按此顺序使用，不要反过来）
情况用什么原因
Tavily 搜得到的信息✅ Tavily返回摘要 snippet，Token 消耗低，速度快
Tavily 搜不到、但页面里有✅ CDP（web-access）必须用浏览器打开才能看到 JS 渲染后的内容
Tavily 搜不到、页面也进不去❌ 放弃或找替代源如需登录才能看的小红书帖子详情，直接放弃
💡 典型场景判断
  • 查新闻/行业报告 → Tavily
  • 官网是 JS 渲染（如很多后台管理页面）→ CDP（web-access）
  • 小红书/抖音详情页 → 放弃（Tavily 搜摘要即可）
  • 需要登录的个人内容 → 放弃
  • 微博/贴吧公开内容 → Tavily（通常有搜索索引）

八、常见问题处理

Q1：Tavily 搜索返回 Unauthorized

原因：API Key 过期、无效或格式不对
解决：到 tavily.com 重新获取 Key，更新 ~/.openclaw/.env，用 Authorization: Bearer 而非 tv-api-key Header

Q2：SSH 隧道访问 OpenClaw 控制台（局域网另一台电脑）

问题：浏览器报错 control ui requires device identity (use HTTPS or localhost secure context)
解决：在远程电脑执行： 
ssh -L 18789:localhost:18789 omi@192.168.1.112
# 然后浏览器访问 http://localhost:18789

Q3：OpenClaw 节点连接报 pairing required

原因：新设备在排队等待批准
解决： 
openclaw devices list       # 查看待批准列表
openclaw devices approve --latest   # 批准最新请求

Q4：PDF 颜色/字体没有生效

原因：Google Fonts 在 PDF 转换环境无法加载
解决：使用内嵌 CSS 样式，避免外部字体依赖（参考本手册 HTML 版本的样式写法）

Q5：Tavily 搜索结果不够详细

原因：Tavily 默认返回摘要snippet，详细内容需要访问原始链接
解决：用 CDP（web-access skill）打开 Tavily 返回的 URL，抓取完整内容
